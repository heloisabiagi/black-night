1. Starting point - Installing, Files and Folders

So, you've just unzipped your brand new Black Night, and you've got a bunch of files and folders: some .html files, a file named package.json, another file named Gruntfile.js, and folders config, src and build. The .html files contains the theme demos, and both the package.json and the Gruntfile.js are destinated to the clients who'll use the Front-End task automator Grunt.js. If you don't know a thing about Grunt.js, no problem, just skip the next lines.

If you intend to use Grunt.js, you probably know what to do next: you need to install the modules. They're specified at the package.json file and if you use npm, all you have to do is:

npm i

And you're now enabled to use the Grunt.js tasks. For further Grunt.js instructions, read the The Gruntfile.js section.

Moving on, the folders: the config contains a json file with settings for those who will use Grunt.js. Then, we have the src, with the not compiled versions of SASS and Javascript files. If you're a developer who knows how to work with Grunt.js and SASS, this folder contains the source files that will be later compiled, concatenated and minified. And last, we have the build folder, which contains the final versions of the files (compiled and minified versions of SASS and Javascript files) that are referenced in the .html files, as well as the images and fonts used in this layout (though the src folder also contains copies of the images and fonts, the .html files point to the ones in the build folder).
