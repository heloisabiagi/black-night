Black Night - Documentation

Thanks for purchasing Black Night. This document will provide usage instructions so you can enjoy Black Night theme at its best. If you find any errors or bugs, please contact us here.

1. Starting point

First, let's start with the files within the theme folder and how to use them.

Once you've downloaded and unzipped the theme, you'll find a bunch of files and folders. The main HTML files are the demo-[type].html, which contain the source code for the theme's pages and can be customized with your own data. The assets folder contains the final concatenated and minified styles, scripts, fonts, images and other sources referenced in the HTML files, while the source folder contains the original files (SASS files, CSS libraries, Javascript libraries, plugins and separate files etc) used for development purposes.

If you intend to use Grunt., you have to install the modules listed in the package.json file. To command line and npm users, it's a pretty simple process: 

npm i

A new folder, node_modules, will be installed in the theme's directory.

For SASS/Compass users, the config.rb file contains all the necessary data to compile Black Night’s files.

Next, let's have a look at the directories in order to understand how to make a better use of Black Night’s functionalities.

1.2 Files and Folders

As mentioned before, besides the HTML files, theme Black Night has two major folders where files are stored: source and assets.

The assets folder contains the final sources used in the theme. It’s the only folder that needs to be put in production along the HTML files. All the external files referenced in the HTML files point to this folder, and this is the only folder that needs to be put in production. This folder is subdivided in 5 directories: style, js, images, fonts and inc.

Black Night theme makes use of SASS/Compass, so, the CSS files inside the assets/style folder are the compiled versions of the files from the source folder. If you’re not using SASS/Compass, you must edit the assets files directly in order to perform changes. The libs.css , and its minified version libs.min.css (which is the one used in the HTML file) contain the third-party libraries from the source/css-libs folder (Bootstrap included), style-[color].css and style-[color].min.css (again, the theme uses the minified versions) files contain the main CSS files (for more information, check item 2.1 Color Schemes), and other CSS files (color-switcher.css, documentation.css and preloader.css) are separated files for specific areas.

The assets/js folder is similar to the style folder: it also contains processed files from the source folder (and if you're not using Grunt to compile tehse files, you ust edit the files in the assets folder), being libs.js and libs.min.js the files with the third-party libraries and plugins generated from the source/js-libs, the scripts.js and scripts.min.js the ones with the theme’s settings, and the other js files (color-switcher.js, modernizr.custom.js and settings.js ) for specific settings (check item 2.4 Javascript Settings).

Both folders assets/images and assets/fonts contain the corresponding file types, while the assets/inc folder contains some other miscellaneous files.

The source folder contains the original development files, and it’s targeted for SASS and Grunt.js users. Inside this folder, you’ll find the following subdirectories: images, fonts and inc (which are actually only safe copies from the ones in the assets equivalent folders), css-libs (which, as mentioned before, contains the original and separated third-party CSS libraries), js-libs (same as css-libs, with Javascript and jQuery plugins), and the folders js and sass, where the actual development files are stored. You can edit these files and then use SASS/Compass and Grunt (check item 1.3 Working with Grunt to see the available tasks) to recompile them in the assets folder.

1.3 Working wth Grunt

If you’re a Grunt user, Black Night makes use of this powerful tasks automator to build its files. The grunt watch task will check for changes in the files from the source folder and process them correctly in the corresponding assets file, so, it’s pretty handy since you don’t have to worry about which files to compile to where. Black Night has also some custom tasks to manually build files:

  • build: the default task that compiles development, settings and third-party scripts and styles
  • devJs: manages changes to the development Javascript files located in source/js
  • devJsLibs: if you need to add new plugins to the source/js-libs directory, use this setting to recreate the assets/js/libs.js and assets/js/libs.min.js files
  • devCSS: manages changes to the SCSS files inside source/sass directory and rebuild them as CSS files to the Code>assets/style folder
  • devCSSLibs: concatenates third-party CSS Libraries from the source/css-libs and generates unique files, assets/style/libs.css and assets/style/libs.min.css

2. Theme features

In the next steps, we will have a look at the theme's features and facilities.

2.1 General Instructions

As mentioned before, the main theme files are the demo-[type].html files and the assets folder, which contains all the CSS, Javascript files, images, fonts and extra files used in this theme.

Once ypu open any of the HTML demo files, you'll notice they're heavily commented. These comments contain usage instructions for the layout and tags, and it's pretty important to read them first before making any changes to the layout in order to prevent malfunction or unexpected behaviors.

If you have any doubts don't hesitate to contact us.

2.2 Color schemes

Black Night has 5 built in color schemes ready to use: blue, green, orange, pink and purple. In order to change the theme color, you have to locate the default stylesheet (blue) in your HTML file:

<link rel="stylesheet" type="text/css" href="assets/style/style-blue.min.css">

And change it for the desired color scheme:

<link rel="stylesheet" type="text/css" href="assets/style/style-pink.min.css">

If you're using SASS/Compass, you can create infinite possibilities of color schemes for your site. All you need to do is to go to the source/sass folder, open any of the style-*.scss files, save it again with the desired color name and then, inside the file, replace the $main-color variable with your new desired color. The variations (darker and lighter) are automatically generated, but you still can set them manually if you wish by changing the $darker-main-color and $lighter-main-color variables with new values. Once you compile this brand new file (that will be generated at the assets folder, don't forget to replace the stylesheet code in your HTML file.

2.3 Fonts and Icons

Black Night is powered by two Google Fonts: Raleway (main font) and News Cycle (headers and buttons). If you wish to use different Google Fonts, you have to replace the following tags: 

<link href='http://fonts.googleapis.com/css?family=News+Cycle:400,700&subset=latin,latin-ext' rel='stylesheet' type='text/css'>

<link href='http://fonts.googleapis.com/css?family=Raleway:400,300,500,700&subset=latin,latin-ext' rel='stylesheet' type='text/css'>

And then replace the fonts in the CSS files. If you're using SASS/Compass, all you have to do is to open the source/sass/settings/_settings.scss file, replace the $base-font and $special-font variables and recompile the files.

As for the icons, Black Night supports two icon font libraries: Glyphicon (default to Bootstrap) and Font Awesome 4.5. Some usage examples:

<span class="icon"><i class="glyphicon glyphicon-menu-right" aria-hidden="true"></i></span>

(Glyphicon example. Note the icon span before the icon itself).

<span class="icon color-icon"><i class="fa fa-quote-left"></i></span>

(Font Awesome example)

2.4 Plugins

In order to enhance user experience, Black Night makes use of the following Javascript plugins:

  • ScrollTo: makes scroll navigation easier.
  • Isotope: a fancy way to display and filter data. Used in the Portfolio section.
  • Magnific Popup: responsive lightbox plugin. Also used in the Portfolio section.
  • Owl Carousel: responsive carousel. You can find it in action at the Testimonials section.
  • Wow: a plugin to reveal animations during scrolling.

The final compiled version of these libraries (libs.js and libs.min.js) can be found at the assets folder, while the original individual files are located at the source/js-libs directory.

2.5 Javascript Settings

For those who do not feel comfortable with opening a Javascript file and editting messages or changing settings, Black Night offers a settings.js file where you can customize things like animations on mobile devices, carousel items, portfolio grid or form feedack messages. If you're working with Grunt, you can edit the file in the source directory; if not, you should directly edit the one in the assets folder.

If you need to add any extra code to Black Night's scripts, you can use the userCustoms module (Black Night uses revealing module pattern for development. You don't need to know design patterns to work with Black Night though) and place it where the message "You can insert customized code right here" stands. For Grunt users, the module is located inside the source/js/components folder; else, it can be found at assets/js/scripts.js and assets/js/scripts.min.js.

2.6 Contact Form

Black Night has a working Contact Form, and customizing it is pretty simple: all you have to do is to open the assets/inc/mailer.php file and change the $to variable name to the desired email.

For Grunt users, if you use the grunt watch task and edit the original source file located at source/inc, it will automatically create a new copy at assets/inc.

2.7 Image Backgrounds

Some sections (like Quote and Newsletter) contain images as backgrounds. If you wish to add background to any section, you just have to add the class image-box to the desired section, and then define the background-image in a new class. For example:

// HTML
<section id="quote-section" class="image-box box01">

// CSS
.image-box.box01 {
  background-image: url('../images/creative-commons/german.jpg?1456034411');
}

The example above is taken from the Quote section, where besides the image-box class, another class containing the background-image data (.box01) was added to the section. The classes box-01 and box-02 are already in use for the 'Quotes' and 'Newsletter' sections respectively (if you wish to change these section's backgrounds, just edit these classes), however, you can create new classes starting with box-03 (in the assets/style-[color].css and assets/style-[color].min.css, you'll notice that the classes .box-03-05 are already created, with empty image background to make it easy to set up).

For SASS/Compass users, filling the new box classes is pretty simple: just open the source/settings/settings.scss files and edit/fill the $image-box-[number] variables.
