Black Night - Documentation

Thanks for purchasing Black Night, a theme built on the Bootstrap framework. This document will provide usage instructions so you can enjoy Black Night at its best. If you find any errors or bugs, you may contact us directly on our WrapBootstrap's user page.

1. Starting point

First, let's start with the files in the theme folder and how to use them.

Once you've downloaded and unzipped the theme, you'll find a set of files and folders. The main HTML files are the ones named demo-[type].html, which contain the source code for the theme's pages and can be customized with your own data. The assets folder contains the final concatenated and minified styles, scripts, fonts, images and other sources referenced in the HTML files, while the source folder contains the original files (SASS files, CSS libraries, Javascript libraries, plugins and separate files etc) used for development.

In case you intend to use Grunt, you'll need to install the modules listed in the package.json file. For command line and npm users, it's a pretty simple process: 

npm i

A new folder, node_modules, will be installed in the theme's directory.

For SASS/Compass users, the config.rb file contains all the necessary data to compile Black Night’s files. Grunt users will find predefined tasks in Gruntfile.js (see 1.3 Working with Grunt for more instructions).

Next, let's have a look at the directories in order to understand how to make a better use of Black Night’s functionalities.

1.2 Files and Folders

As mentioned before, in addition to the HTML files and other specific settings files (package.json, config.rb and Gruntfile.js), Black Night has two major folders where files are stored: source and assets. The files in the source folder are development files which can be edited and processed with SASS/Compass and Grunt, while the assets files are the final compiled files, so, if you're not using any development processing tool (like SASS/Compass or Grunt), this is your folder.

1.2.1 Assets Folder - editting theme files

The assets folder contains the final sources used in the theme. It’s the only folder that needs to be put in production along with the HTML files. All the external files referenced in the HTML files point to this folder, which is is subdivided in 5 directories: style, js, images, fonts and inc.

Black Night is built on SASS/Compass frameworks , so, the CSS files inside the assets/style folder are the compiled versions of the files from the source folder. If you’re not using SASS/Compass, you must directly edit the assets files in order to perform changes to the CSS files. The libs.css , and its minified version libs.min.css contain the third-party libraries from the source/css-libs folder (Bootstrap included), while style-[color].css and style-[color].min.css files contain the main CSS files (for more information, check item 2.1 Color Schemes). The CSS files referenced in the HTML files are the .min.css ones, so, you can either edit them directly or edit the original .css files and minify them later.

The assets/js folder is similar to the style folder: it contains processed files from the source folder (if you're not using Grunt to compile these files, you must edit them in the assets folder): libs.js and libs.min.js are the files with the third-party libraries and plugins generated from the source/js-libs, while the scripts.js and scripts.min.js are the ones with the theme’s settings, and , finally, the other js files (modernizr.custom.js and settings.js ) are used for specific settings (check item 2.4 Javascript Settings). Just like the CSS files, Black Night uses the .min.js files in the HTML files, so you can edit the original .js files and minify them later.

Both folders assets/images and assets/fonts contain the corresponding file types, while the assets/inc folder contains other miscellaneous files.

1.2.2 Source Folder - development files

The source folder contains the original development files, and it’s targeted for SASS and Grunt.js users. Inside this folder, you’ll find the following subdirectories: images, fonts and inc (which are actually just safe copies from the ones in the assets equivalent folders), css-libs (which, as mentioned before, contains the original and separated third-party CSS libraries), js-libs (same as css-libs, with Javascript and jQuery plugins), and the folders js and sass, where the actual development files are stored. You can edit these files and then use SASS/Compass and Grunt (check item 1.3 Working with Grunt to see the available tasks) to recompile them in the assets folder.

1.3 Working with Grunt

If you’re a Grunt user, know that Black Night makes use of this powerful tasks runner to build its files. The grunt watch task will check for changes in the files from the source folder and process them correctly in the corresponding assets file, so, it’s pretty handy since you don’t have to worry about which files to compile and where to put them. Black Night has also some custom tasks to manually build files:

  • build: the default task that compiles development, settings and third-party scripts and styles
  • devJs: manages changes to the development Javascript files located in source/js
  • devJsLibs: if you need to add new plugins to the source/js-libs directory, use this task to recreate the assets/js/libs.js and assets/js/libs.min.js files
  • devCSS: manages changes to the SCSS files inside source/sass directory and rebuilds them as CSS files to the assets/style folder
  • devCSSLibs: concatenates third-party CSS Libraries from the source/css-libs and generates unique files, assets/style/libs.css and assets/style/libs.min.css

2. Theme features

In the following steps, we will have a look at the theme's features and facilities.

2.1 General Instructions

As mentioned before, the main theme files are the demo-[type].html files and the assets folder, which contains all the CSS, Javascript files, images, fonts and extra files used in this theme.

Once you've opened any of the HTML demo files, you'll notice they're heavily commented. These comments contain usage instructions for the layout and tags (like classes usages, tags start and end, character limits etc), and it's pretty important to read them first before making any changes to the layout in order to prevent malfunction or unexpected behaviors.

If you have any doubts don't hesitate to contact us.

2.2 Color schemes

Black Night has 5 built-in color schemes ready to use: blue, green, orange, pink and purple. In order to change the theme color, you have to locate the default stylesheet tag (blue) in your HTML file:

<link rel="stylesheet" type="text/css" href="assets/style/style-blue.min.css">

And replace it with the desired color scheme:

<link rel="stylesheet" type="text/css" href="assets/style/style-pink.min.css">

If you're using SASS/Compass, you can create countless possibilities of color schemes for your site. All you need to do is to go to the source/sass folder, open style-blue.scss file, save it again with the desired color name (i.e style-aqua.scss) and then, inside the file, replace the $main-color variable with your new desired color. The variations (darker and lighter) are automatically generated, but you can still set them manually if you wish, by replacing the $darker-main-color and $lighter-main-color variables with new values. Once you've compiled this brand new file (that will be generated in the assets folder), don't forget to replace the stylesheet code in your HTML file.

2.3 Fonts and Icons

Black Night is powered by two Google Fonts: Raleway (main font) and News Cycle (headers and buttons). If you wish to use different Google Fonts, you have to replace the following tags: 

<link href='http://fonts.googleapis.com/css?family=News+Cycle:400,700&subset=latin,latin-ext' rel='stylesheet' type='text/css'>

<link href='http://fonts.googleapis.com/css?family=Raleway:400,300,500,700&subset=latin,latin-ext' rel='stylesheet' type='text/css'>

Then, replace the fonts in the CSS files. If you're using SASS/Compass, all you'll have to do is open the source/sass/settings/_settings.scss file, replace the $base-font and $special-font variables and recompile the files.

As for the icons, Black Night supports two icon font libraries: Glyphicons (default to Bootstrap) and Font Awesome 4.5. Some usage examples:

<span class="icon"><i class="glyphicon glyphicon-menu-right" aria-hidden="true"></i></span>

(Glyphicons example. Note the icon span before the icon itself).

<span class="icon color-icon"><i class="fa fa-quote-left"></i></span>

(Font Awesome example)

2.4 Plugins

In order to enhance user's experience, Black Night makes use of the following Javascript plugins:

  • ScrollTo: makes scroll navigation easier.
  • Isotope: a fancy way to display and filter data. Used in the Portfolio section.
  • Magnific Popup: responsive lightbox plugin. Also used in the Portfolio section.
  • Owl Carousel: responsive carousel. You can find it in action at the Testimonials section.
  • jQuery Validate: an easy-to-use jQuery validation plugin.
  • Wow: a plugin to reveal animations during scrolling.

The final compiled version of these libraries (libs.js and libs.min.js) can be found at the assets folder, while the original individual files are located at the source/js-libs directory.

2.5 Javascript Settings

For those who do not feel comfortable with opening a Javascript file and editing messages or changing settings, Black Night offers a settings.js file where you can customize things like animations on mobile devices, carousel items, portfolio grid or form feedback messages. If you're working with Grunt, you can edit the file in the source directory; if not, you should directly edit the one in the assets folder.

If you need to add any extra code to Black Night's scripts, you can use the userCustoms module and place it where the message "You can insert customized code right here" stands (Black Night uses revealing module pattern for development. You don't need to know design patterns to work with Black Night though). For Grunt users, the module is located inside the source/js/components folder; else, it can be found at assets/js/scripts.js and assets/js/scripts.min.js.

2.6 Contact Form

Black Night has a working Contact Form, and customizing it is pretty simple: all you have to do is open the assets/inc/mailer.php file and change the $to variable name to the desired email.

For Grunt users, if you use grunt watch task and edit the original source file located at source/inc, it will automatically create a new copy at assets/inc.

2.7 Image Backgrounds

Some sections (like Quote and Newsletter) contain images as backgrounds, which are declared directly in the HTML file. If you wish to add background to any section, you just need to add the class image-box to the desired section, and then add the background-image in a style declaration. For example:

<section id="services" class="image-box" style="background-image: url(assets/images/[your-folder]/[your-image.jpg]);">
